# Contributing

## Development

```bash
make setup      # one-time: download deps + install tooling (lint, releaser)
make demo       # build, then validate + compile the example into ./dist
make build      # build ./bin/omniplug
make test       # run tests
make check      # fmt + vet + test (pre-commit gate)
make cover      # coverage summary
make lint       # golangci-lint
```

To work on the docs site (needs Hugo extended — `brew install hugo`):

```bash
make docs-serve   # regenerate the CLI reference + live preview at http://localhost:1313
make docs         # build the static site into site/public
```

The site lives under `site/` (Hugo + the hugo-book theme). Prose pages are in `site/content/docs/`; the CLI reference under `site/content/docs/reference/` is generated by `omniplug gen-docs` and is not committed.


## Project layout

```text
cmd/omniplug/        entrypoint (registers adapters via blank import)
internal/model/      canonical IR
internal/parser/     source -> IR
internal/schema/     validation
internal/adapter/    Adapter interface + registry
internal/adapters/   one package per target (claude, cursor, ...)
internal/yamlfm/     shared YAML frontmatter builder
internal/compiler/   orchestration over the registry
internal/installer/  filesystem placement + dry-run
internal/cli/        cobra commands
examples/            sample canonical plugins
```

## Adding a new target

The core is closed for modification, open for extension. To add a target (e.g. Codex or Grok):

1. Create `internal/adapters/<name>/` implementing `adapter.Adapter` — `Name()`, `Capabilities()`, `Validate()`, `Compile()` (pure: IR → files), and `InstallPlan()`.
2. Declare `Capabilities()` so the compiler can degrade unsupported components with diagnostics instead of emitting wrong output.
3. Register it: `func init() { adapter.Register(&Adapter{}) }`, and add a blank import in `cmd/omniplug/main.go`.
4. Add golden/unit tests under the adapter package.

No edits to the parser, compiler, installer, or CLI are required — that is the test of whether the abstraction holds.

## Releasing

Releases are tag-driven:

```bash
git tag v0.1.0
git push origin v0.1.0
```

This triggers `.github/workflows/release.yml`, which runs GoReleaser (cross-platform binaries, checksums, the Homebrew formula) and publishes the npm wrapper. See [`.goreleaser.yaml`](https://github.com/asingamaneni/omniplug/blob/main/.goreleaser.yaml) for the required `HOMEBREW_TAP_TOKEN` and `NPM_TOKEN` secrets.

GoReleaser isn't installed by `make setup` (it's release-only). Install it when needed:

```bash
brew install goreleaser            # or: go install github.com/goreleaser/goreleaser/v2@latest
make release-check                 # goreleaser check — validate the config
make snapshot                      # local build of all artifacts into ./dist
```
