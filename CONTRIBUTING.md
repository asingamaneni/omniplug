# Contributing to omniplug

Thanks for your interest in contributing! Issues and pull requests are welcome.

## Quick start

```bash
git clone https://github.com/asingamaneni/omniplug
cd omniplug
make setup      # one-time: download deps + install tooling (lint, hugo)
make demo       # build, then validate + compile the example into ./dist
make check      # fmt + vet + test (pre-commit gate)
```

Requires Go 1.23+.

## Making changes

1. Fork and create a topic branch.
2. Make your change, with tests. `make check` must pass; golden-file tests
   pin adapter output byte-for-byte (`go test ./internal/adapters/... -update`
   regenerates them — include golden changes in the same commit and explain
   them in the PR).
3. Open a pull request describing what changed and why.

## Adding a target adapter

The core design promise is that new targets need one package and zero core
changes:

1. Create `internal/adapters/<name>/` implementing `adapter.Adapter`.
2. Declare `Capabilities()`, implement `Compile()` (pure: IR → files) and
   `InstallPlan()`.
3. `func init() { adapter.Register(&Adapter{}) }` plus a blank import in
   `cmd/omniplug/main.go`.
4. Add golden tests against `examples/hello-plugin` (see the existing
   `golden_test.go` files and `internal/adapters/goldentest`).

Where a canonical field has no native home on your target, **degrade it with a
diagnostic** — never emit silently-wrong output. Cite the official docs you
validated the output format against in your PR.

## Docs site

The site lives under `site/` (Hugo + hugo-book, vendored). Prose pages are in
`site/content/docs/`; the CLI reference is generated by `omniplug gen-docs`
and is not committed.

```bash
make docs-serve   # live preview at http://localhost:1313
```
